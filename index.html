01 · How it works

From export to insight, in three moves.

You already download statements. That's the whole habit this app asks of you — the automaton handles everything after the file lands.

STEP 01

Export what you already have

Download statements the way you always do — CSV from the bank, Excel from the broker. No new habits, no browser extensions, no linked logins.

STEP 02

Drop it in

The importer detects what each file is, parses it, skips anything it has seen before, and categorizes every row on arrival.

bank transactionsbroker holdingsbroker trades
STEP 03

Read the picture

Net worth by bucket, monthly cash flow, top spending, income sources, asset allocation — plus a snapshot history that builds itself every time you visit.

02 · Features

Everything gets a place.

Built like an accountant, runs like a machine — every row filed, every account bucketed, every month projected.

Drop-anything import

Bank CSV or broker Excel — the importer works out whether a file holds transactions, holdings, or trades, then parses and files every row accordingly.

Import twice, stored once

Every row is content-hashed, so re-uploading the same or overlapping statements is always safe — duplicates are recognized and skipped, not double-counted.

Rules you can read

Spending is categorized by plain keyword rules — "WHOLEFDS" → Groceries — with 19 sensible defaults. Edit a rule or reassign any row whenever you like. No black box.

A net worth with memory

Cash, taxable, tax-free, retirement, real estate, liabilities — bucketed and totalled, with snapshots captured automatically so your trend line draws itself. Untracked assets like a home take a manual value.

Holdings, seen whole

Cost basis, market value, and unrealized gain for every position — equities, ETFs, funds, bonds, crypto — rolled into an allocation view across all your brokers at once.

Budgets that look forward

Set monthly targets per category — or auto-fill them from your own averages — then project income, spending, and cumulative savings 3 to 12 months out.

03 · Privacy

Your bank password is none of our business.

There is nothing to link, nothing to scrape, and no aggregator in the middle. Financial Automata reads the files you choose to give it, on hardware you own.

The usual way
Your bank your password Aggregator cloud your balances Someone's app
✕ Your credentials live on servers you've never met.
The Automata way
Your bank a file you download Your machine parsed & filed Your Postgres
✓ Credentials never leave the bank. Data never leaves your machine.

Self-hosted, end to end

FastAPI backend, React frontend, and a PostgreSQL database that sits on your own disk. There's no cloud tier because there's no cloud.

Multi-user, hard walls

Accounts are isolated per user — every query is scoped, and another user's data simply doesn't exist as far as yours is concerned.

Nothing decided in secret

Categorization is a list of keyword rules you can open, read, and change — not a model you have to trust.

04 · Get started

Running on your machine in one evening.

Three processes, one database, and a folder of sample statements to try before you commit your own.

quickstart — zsh
# needs python 3.11+ (uv), node 18+, postgresql 15
$ createdb financial_planner

# api on :8000 — interactive docs at /docs
$ cd backend && uv run uvicorn app.main:app --reload

# app on :5173
$ cd frontend && npm run dev
  ➜  Local: http://localhost:5173/
